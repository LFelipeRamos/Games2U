# 1. Requisitos Não Funcionais

<p align="justify">
A <i>Tabela 2</i> a seguir contém os Requisitos Não Funcionais (RNF), que definem as restrições de qualidade do sistema Games2U.
</p>

| ID    | Requisito                                                                 | Categoria        |
| :---: | ------------------------------------------------------------------------- | ---------------- |
| RNF01 | O sistema deve ser responsivo e funcionar em dispositivos móveis e desktop.| Usabilidade     |
| RNF02 | O tempo de resposta das requisições deve ser inferior a 3 segundos.      | Desempenho       |
| RNF03 | O sistema deve garantir segurança na autenticação (criptografia de senha).| Segurança        |
| RNF04 | O sistema deve utilizar HTTPS para comunicação segura.                   | Segurança        |
| RNF05 | O sistema deve suportar múltiplos usuários simultâneos.                 | Escalabilidade   |
| RNF06 | O sistema deve estar disponível 99% do tempo.                             | Disponibilidade  |
| RNF07 | O sistema deve ser desenvolvido com NextJS e TypeScript.                | Tecnológico      |
| RNF08 | O sistema deve possuir interface intuitiva e amigável.                  | Usabilidade      |
| RNF09 | O sistema deve permitir fácil manutenção e evolução.                    | Manutenibilidade |
| RNF10 | O sistema deve registrar logs de operações importantes.                 | Auditoria        |
| RNF11 | O sistema deve estar em conformidade com a LGPD.                        | Legal            |
| RNF12 | O sistema deve integrar com gateways de pagamento confiáveis.           | Integração       |
